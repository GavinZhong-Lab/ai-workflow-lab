# 阅读器模块 — 业务逻辑

## 模块概述

XC-AiReader 是 AI Workflow Lab 平台内置的小说阅读应用，支持小说浏览、在线阅读、收藏、阅读进度跟踪、多语言翻译、爬虫自动采集和后台管理。

## 用户角色

| 角色 | 权限 |
|------|------|
| 普通用户 | 浏览/搜索小说、阅读章节、收藏/取消收藏、阅读进度、翻译 |
| 超级管理员 | 以上所有 + 管理后台（小说/章节/横幅/翻译配置/爬虫） |

## 核心业务流程

### 1. 小说浏览

```
进入阅读器 → 加载 Banner → 加载小说列表（分页）
  → 搜索（标题/作者模糊匹配）
  → 分类过滤（玄幻/仙侠/都市/历史/科幻/游戏/悬疑/轻小说/短篇）
  → 全部小说 / 我的收藏 切换
  → 继续阅读卡片（有进度的小说）
  → 点击进入详情页
```

### 2. 在线阅读

```
详情页 → 点击章节 → 进入阅读视图
  → 顶栏固定（返回按钮 + 章节信息）
  → 正文区域滚动
  → 滚动进度跟踪（2秒防抖保存）
  → 底栏浮动（上一章/目录/下一章）
  → 章节目录抽屉（右侧滑出，点击跳转）
  → 语言切换（中文/EN/日本語/Русский/Français/Español）
```

### 3. 翻译流程

```
点击语言切换 → 选择非中文语言
  → 按段落分割正文 → 按2500字符分组
  → 调用 POST /api/v1/reader/translate/batch（顺序翻译，200ms间隔）
  → 翻译完成替换正文显示（保留段落结构）
  → 切回中文恢复原文
  → 翻译结果按章节缓存
```

### 4. 收藏流程

```
浏览页/详情页 → 点击收藏按钮
  → 已收藏：调用 DELETE /api/v1/reader/favorites/:novelId
  → 未收藏：调用 POST /api/v1/reader/favorites
  → 实时更新 UI 状态
```

### 5. 阅读进度

```
进入章节 → 若首次打开且无进度，保存0%
  → 滚动时2秒防抖计算百分比
  → 调用 POST /api/v1/reader/progress
  → 切换章节/卸载时 flush 最终进度
  → 浏览页/详情页展示继续阅读入口
```

### 6. 爬虫采集（管理员）

```
后台配置爬虫参数 → 启动 Puppeteer 浏览器
  → 获取小说列表 → 去重检查（sourceUrl / title+author）
  → 获取章节目录（上限10000章）
  → 逐章采集内容（3-8秒随机延迟）
  → 存储到 Novel + Chapter 表
  → 支持断点续爬（INCOMPLETE 状态恢复）
  → 支持优雅停止（完成当前章节后停止）
  → 实时状态监控（当前小说/章节/进度）
```

## 业务规则

- 小说列表仅展示 `isActive: true` 的小说
- 章节数上限 10000 章（防止异常数据）
- 翻译每条 ≤3000 字符，每次批量 ≤20 条
- 翻译按顺序执行，200ms 间隔防止限流
- 爬虫去重规则：sourceUrl 精确匹配 或 title+author 组合匹配
- 爬虫支持断点续爬：INCOMPLETE 状态小说在下次爬虫启动时自动恢复
- 收藏去重：同一用户对同一小说只能收藏一次
- 阅读进度 upsert：同一用户对同一小说只有一条进度记录
