## 架构概览

```

  page.tsx                         # 轻量路由组件（~80行）
  hooks/use-reader-data.ts         # 数据/状态/操作 hook（~160行）
  hooks/use-translated-texts.ts   # 翻译 hook (单条/批量)
  components/

    browse-view.tsx                # 浏览视图

    reader-view.tsx                # 阅读视图
    chapter-directory.tsx          # 章节目录抽屉

  reader.schema.ts                 # Zod 校验
  reader.service.ts                # 公开 + 管理业务逻辑
  reader.controller.ts             # 公开 API 控制器
  reader.routes.ts                 # 公开路由注册

  reader-admin.routes.ts           # 管理路由注册
```

## 关键实现细节

### 翻译批量接口
- 后端使用原生数组批量 API（DeepL/Google/Microsoft），单次请求完成全部翻译
- 通过 `EnvHttpProxyAgent` (undici) 自动适配系统代理环境变量（HTTP_PROXY/HTTPS_PROXY）
- 单条 ≤3000 字符，单次 ≤20 条
- 前端按段落分割 + 2500 字符分组确保不超限

### 全局翻译状态管理
- Zustand store (`useTranslationStore`) + `persist`中间件持久化语言偏好
- 内容寻址缓存：`normalize(text) + targetLang` 作为 key，同文不同上下文共享翻译
- 100ms 防抖批量入队，自动去重，跳过 [Mock 结果不缓存
- `useTranslatedText` / `useTranslatedTexts` hooks 自动检测未缓存文本并入队
- 章节正文独立缓存：`chapterId:content:targetLang`
- 切换语言自动清空内存缓存，保留语言偏好到 localStorage

### 阅读进度跟踪
- 滚动计算 `scrollTop / (scrollHeight - clientHeight) * 100`
- 2 秒防抖后调用 API 保存
- 切换章节/卸载时 flush 最终进度
- 首次打开无进度时保存 0%（不覆盖已有进度）

### 爬虫断点续爬
- 每次启动优先恢复 INCOMPLETE 状态的小说
- 从 `lastChapterIndex` 继续采集
- 支持优雅停止（设置 isStopping 标志位，完成当前章节后停止）
- `resumeNovel` 将 FAILED 改为 INCOMPLETE 等待下次爬虫启动

### 前端组件拆分
- `page.tsx` 仅负责根据 view 状态渲染子组件（~84行）
- `use-reader-data.ts` hook 封装所有状态和数据操作
- 每个视图组件职责单一，通过 props 通信

### i18n 实现
- `useTranslations('reader')` 读取 zh-CN.json / en-US.json 中的 `reader.*` key
- 管理后台新增 40+ 翻译 key（reader.admin.*）
- Sidebar "Reader Admin" 使用翻译 key
