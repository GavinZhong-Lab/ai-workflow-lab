# AGENTS.md — AI 助手行为规则

## 核心原则

1. **文档先行** — 所有开发任务必须先输出设计方案文档
2. **按模块归档** — 文档按功能模块存放到 `/docs/{模块名}/` 目录
3. **等待审查** — 设计方案输出后必须等待用户审查确认
4. **审查通过后编码** — 用户确认「方案通过」或「可以开始」后才能编码

## 工作流

```
用户提出需求
    ↓
分析需求，确定所属功能模块
    ↓
在 /docs/{模块名}/ 下创建设计文档（业务逻辑.md / 技术方案.md / 数据模型.md）
    ↓
等待用户审查确认
    ↓
用户确认后 → 开始编码实现
    ↓
编码完成 → 更新文档和进度
```

## 文档规范

- 每个功能模块一个独立目录
- 每个模块包含：业务逻辑.md、技术方案.md、数据模型.md
- 文档使用 Markdown 格式
- 文档需包含清晰的标题层级和目录结构

## 代码规范

- **注释要求** — 每个文件顶部须有文件说明注释，每个类和方法须有功能注释
- **类型安全** — 前后端共享 TypeScript 类型（`packages/shared`）
- **模块结构** — Express 模块按 controller/service/schema/routes 拆分
- **前端组件** — React 组件使用 `'use client'` 指令，服务端组件默认

## 项目结构

```
saas-platform/
├── apps/api/       # 后端 Express API (端口 4000)
├── apps/web/       # 前端 Next.js (端口 3000)
├── packages/shared/ # 共享类型与常量
├── docs/           # 设计文档
└── docker-compose.yml  # PostgreSQL + Redis + MinIO
```

## 快捷指令

| 指令 | 行为 |
|------|------|
| 「保存进度」 | 更新 CLAUDE.md 进度 + 更新进度追踪文档 + 列出未提交更改 |
| 「继续开发」 | 读取 CLAUDE.md + 进度追踪 + 决策记录 + git log，总结状态 |
| 「项目状态」 | 显示当前项目进度、最近变更、待办事项 |

## 前端设计原则

- 避免 AI slop 审美 — 不使用 Inter/Roboto/Arial 等通用字体
- 使用独特配色方案 — 避免紫色渐变配白色背景的陈词滥调
- 优先使用 CSS 动画 — 入场动画、页面切换使用 Framer Motion
- 深色/亮色双主题 — 使用 CSS 变量实现主题切换
- 玻璃态卡片 — 使用 backdrop-filter 营造深度感
